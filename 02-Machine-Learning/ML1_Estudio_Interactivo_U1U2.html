1.1 · CONCEPTO
Concepto y Tipos de ML
▼
Definición ISO (2023): El aprendizaje automático es la disciplina que desarrolla algoritmos capaces de mejorar su desempeño en tareas específicas a partir de datos, sin ser programados explícitamente para cada caso.
Diferencia clave vs programación tradicional: Programación tradicional → el humano escribe las reglas. ML → el algoritmo aprende las reglas a partir de los datos.
🏷️
1. Supervisado
Datos con etiquetas conocidas. El más usado en industria.

Ej: clasificación de imágenes, diagnóstico médico, predicción de precios.
🔍
2. No Supervisado
Sin etiquetas. Descubre estructuras ocultas en los datos.

Ej: segmentación de clientes, análisis de redes sociales, PCA.
🔀
3. Semi-supervisado
Pocos datos etiquetados + muchos sin etiquetar. Útil cuando etiquetar es costoso.

Ej: genómica, imágenes médicas.
🎮
4. Por Refuerzo
Agente aprende por recompensas / penalizaciones interactuando con un entorno.

Ej: robótica, videojuegos complejos, trading.
🧠
5. Auto-supervisado
Genera sus propias etiquetas desde los datos. Base de LLMs.

Ej: GPT, BERT, modelos de visión por computadora.
🔗
Sistemas Híbridos
Combinan múltiples paradigmas para arquitecturas más robustas y versátiles. Mayor flexibilidad.

DESAFÍOS CRÍTICOS
01
Sesgos y Equidad
Los algoritmos reproducen sesgos de los datos → retos éticos y sociales (Mersha et al., 2024).
02
Complejidad Computacional
Modelos profundos requieren GPU/TPU y alto consumo energético (Hao et al., 2022).
03
Calidad de Datos
Ruido, valores faltantes y distribuciones no representativas comprometen la validez del modelo (Dritsas et al., 2025). Regla: "Garbage In = Garbage Out".
04
Interpretabilidad
Necesidad de modelos explicables en medicina y finanzas. → Subcampo XAI (Explainable AI) y Physics-Informed ML.

RESUMEN · S1
Conceptos Clave — Lo que debes recordar
▼
  • ML aprende patrones sin código explícito → diferencia fundamental con programación tradicional basada en reglas fijas.
  • Cada tipo de ML tiene un propósito distinto → elige según disponibilidad de etiquetas y tipo de retroalimentación.
  • Los datos son lo más importante → calidad > cantidad. 80% del trabajo en ML es preparación y limpieza de datos.
  • El término "IA" surgió en los 1950s — Conferencia de Dartmouth 1956, John McCarthy.
  • Resurgimiento IA en los 2000s = mayor poder computacional + Big Data.
  • Los 2010s = era del Deep Learning: AlexNet (2012), Transformers (2017), GPT (2018+).
  • Sesgos y ética son fundamentales → XAI busca transparencia en decisiones críticas.
  • En educación, ML adapta contenido pero debe evitar sesgos en recomendaciones personalizadas.
  • En civil/ambiental: ML predice deslizamientos e inundaciones, no traduce textos ni genera multimedia.

1.3 · MATH (S2)
Álgebra Lineal, Probabilidad y Estadística
▼
📐
Álgebra Lineal
→ Datos en vectores y matrices
→ Transformaciones lineales
→ SVD, PCA como descomposiciones clave
→ Álgebra aleatorizada para datos masivos
📊
Probabilidad
→ Incertidumbre mediante distribuciones
→ Procesos estocásticos
→ Modelos de Markov esenciales en refuerzo
→ Teorema de Bayes
📈
Estadística
→ Métodos de inferencia y validación
→ Pruebas de hipótesis
→ Asegurar generalización
→ Evitar patrones espurios (Symeonidis et al., 2022)
Modelos de Markov: Modelan procesos estocásticos y secuenciales. Probabilidad de transición: P(estado siguiente | estado actual). Son la base matemática del Aprendizaje por Refuerzo. NO son datos deterministas ni se limitan al álgebra lineal.
Por qué los 3 pilares juntos son esenciales: Álgebra lineal (representar datos) + Probabilidad (manejar incertidumbre) + Estadística (validar y generalizar) → algoritmos robustos y generalizables.

1.4 · HERRAMIENTAS
Ecosistema Computacional de ML
▼
Python
Lenguaje principal de ML
Simplicidad + extenso ecosistema. Domina sobre R, Java, Scala. Preferido por productividad + comunidad + librerías especializadas.
NumPy
Computación numérica — arreglos multidimensionales
Operaciones matriciales eficientes. Base de todo el ecosistema científico Python. No hace visualización ni modelos directamente.
Pandas
Manipulación de datos tabulares (DataFrames)
Cargar CSV/Excel, limpiar, filtrar, agrupar, pivotar. Series y DataFrames como estructuras. El "Excel" de Python para ML. No crea redes neuronales.
Matplotlib
Visualización de datos
Gráficas, histogramas, scatter plots, reportes visuales. Seaborn lo complementa para visualización estadística. No hace modelado.
Scikit-learn
ML clásico — clasificación, regresión, clustering
Interfaz uniforme: fit() → predict() → score(). Estándar para modelos tradicionales. No hace deep learning.
TensorFlow
Deep Learning — Google
Construcción y entrenamiento de redes neuronales profundas. Keras como API de alto nivel. Orientado a producción a escala.
PyTorch
Deep Learning flexible — Meta
Prototipado y entrenamiento de modelos complejos. Favorito en investigación académica por su dinamismo y facilidad de debug.
Jupyter
Entorno interactivo reproducible
Código + texto + gráficas en un documento. Permite documentar experimentos paso a paso. Estándar en ciencia de datos.
Colab
Jupyter en la nube — Google
GPU/TPU gratis, sin instalación local. Ideal para comenzar proyectos de ML sin hardware especializado.
MLOps
Ingeniería de software + Ciencia de datos
Reproducibilidad y escalabilidad en producción. CI/CD de modelos (Symeonidis et al., 2022).

RESUMEN · S2
Conceptos Clave — Lo que debes recordar
▼
  • Supervisado = pares entrada→salida con etiquetas conocidas. Ejemplo clásico y correcto: filtro de spam.
  • No supervisado = sin etiquetas, descubre estructura oculta. Ejemplo: clustering de clientes.
  • Modelos de Markov = probabilidad aplicada a procesos secuenciales y estocásticos (NO son deterministas, NO solo álgebra lineal).
  • Álgebra lineal = lenguaje de ML. Datos como vectores y matrices. SVD y PCA son descomposiciones clave.
  • Python preferido por productividad + comunidad + librerías, NO porque sea el único con condicionales o no dependa de librerías.
  • Pandas ≠ deep learning. Pandas = DataFrames tabulares. TensorFlow/PyTorch = redes neuronales profundas.
  • Jupyter = documentación reproducible. Colab = Jupyter en la nube con GPU gratis.
  • Los 3 pilares matemáticos juntos permiten diseñar algoritmos robustos y generalizables.
