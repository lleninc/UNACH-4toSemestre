Tema 1
Rol y funciones del DBA
▼

La administración de bases de datos es la disciplina que planifica, organiza y controla los datos almacenados en un SGBD, garantizando disponibilidad, integridad, seguridad y accesibilidad.

Definición: El DBA es el profesional responsable de garantizar que la información sea segura, íntegra y esté disponible. Combina conocimientos técnicos con habilidades para resolver problemas.
  • Instalar y mantener el gestor de base de datos (SGBD)
  • Diseñar estructuras de almacenamiento y organización
  • Administrar usuarios, roles y permisos de acceso
  • Implementar seguridad, respaldos y recuperación de datos
  • Monitorear rendimiento, prevenir fallos y proponer mejoras
⚛️
Tema 2
Transacciones y propiedades ACID
▼

Una transacción es una secuencia de operaciones que se ejecuta como una unidad indivisible. Si algún paso falla, se revierten todas las operaciones (ROLLBACK) para evitar inconsistencias.

Propiedades ACID:

  • Atomicidad (A): Todo se ejecuta o nada se ejecuta.
  • Consistencia (C): Se respetan todas las reglas y restricciones.
  • Aislamiento (I): Las transacciones concurrentes no se interfieren.
  • Durabilidad (D): Los cambios confirmados persisten permanentemente.
Diferencia COMMIT vs ROLLBACK: COMMIT confirma y persiste los cambios. ROLLBACK cancela y deshace todos los cambios de la transacción actual.
🔒
Tema 3
Bloqueos y niveles de aislamiento
▼

Un bloqueo es el mecanismo que usan los SGBD para coordinar el acceso concurrente y evitar inconsistencias.

  • Compartido (S): Permite lectura concurrente por múltiples transacciones.
  • Exclusivo (X): Solo una transacción puede escribir; las demás esperan hasta COMMIT/ROLLBACK.

Niveles de aislamiento (de menor a mayor seguridad):

  • Read Uncommitted: Permite leer datos no confirmados (riesgo: lecturas sucias).
  • Read Committed: Evita lecturas sucias, mejor equilibrio rendimiento/seguridad.
  • Repeatable Read: Mantiene consistencia en lecturas repetidas.
  • Serializable: Nivel más estricto, equivalente a ejecución secuencial.

Tema 4
Índices y sentencias avanzadas
▼

Un índice es una estructura auxiliar que mantiene claves ordenadas para acelerar búsquedas, JOIN, ORDER BY y GROUP BY.

  • Clustered: Define el orden físico. Solo uno por tabla.
  • Non-Clustered: Estructura separada. Varios por tabla.

Sentencias avanzadas:

  • JOIN: Combina tablas (INNER, LEFT, RIGHT, FULL).
  • CTE (WITH): Tablas temporales para organizar consultas complejas.
  • Funciones de ventana (OVER): Cálculos por grupo sin perder detalle.

Práctica 1: Transacciones y Bloqueos
▼

Bloque 1: Crear Base de Datos

IF DB_ID('Vuelos') IS NULL CREATE DATABASE Vuelos; GO USE Vuelos; GO CREATE TABLE Vuelos ( vuelo_id INT PRIMARY KEY, destino VARCHAR(50), asientos_disponibles INT, precio DECIMAL(10,2) ); INSERT INTO Vuelos VALUES (101, 'Galápagos', 2, 150.00), (102, 'Quito', 15, 60.00);

Bloque 2: Transacción con Control de Errores

BEGIN TRANSACTION; BEGIN TRY INSERT INTO Reservas (vuelo_id, pasajero_nombre) VALUES (101, 'Lenin Lopez'); UPDATE Vuelos SET asientos_disponibles = asientos_disponibles - 1 WHERE vuelo_id = 101; COMMIT; END TRY BEGIN CATCH ROLLBACK; END CATCH;

Práctica 2: Índices y Consultas Avanzadas
▼

Índices y JOINs

CREATE NONCLUSTERED INDEX IX_Vuelos_Destino ON Vuelos(destino); SELECT V.destino, R.pasajero_nombre FROM Vuelos V FULL JOIN Reservas R ON V.vuelo_id = R.vuelo_id;

CTE y Funciones de Ventana

WITH PromedioPrecios AS ( SELECT AVG(precio) as precio_medio FROM Vuelos ) SELECT V.destino, V.precio FROM Vuelos V, PromedioPrecios P WHERE V.precio > P.precio_medio; SELECT destino, precio, RANK() OVER (ORDER BY precio DESC) as ranking FROM Vuelos;

Resumen
Conceptos clave para estudiar
▼
  1. Bloqueos y aislamiento protegen consistencia con concurrencia.
  2. Serializable ofrece máxima seguridad con mayor costo.
  3. Los índices aceleran consultas con criterio de diseño.
  4. JOIN y subconsultas resuelven consultas multi-tabla.
  5. CTE y funciones de ventana fortalecen análisis SQL avanzado.
