Conceptos clave de U1 T1 — para repasar antes del examen

Sistemas numéricos
01 // DECIMAL
Sistema base 10
Usa los dígitos 0–9. Un número real tiene signo, parte entera y parte decimal separadas por el punto decimal.
02 // BINARIO
Sistema base 2
Solo usa 0 y 1. Es el lenguaje nativo de las computadoras: representa presencia (1) o ausencia (0) de pulso eléctrico. La unidad mínima es el bit.
03 // BYTE y PALABRA
Agrupación de bits
1 byte = 8 bits. Una "palabra" puede tener 8–64 bits. Una palabra de 32 bits = 4 bytes, de 64 bits = 8 bytes.
Representación de enteros
Magnitud con signo (Sign-Magnitude)
El primer bit (MSB) indica el signo: 0 = positivo, 1 = negativo.
Los bits restantes almacenan la magnitud del número.
Ejemplo: −6 en 8 bits → 6₁₀ = 00000110₂ → con signo negativo: 10000110
Ejemplo: +12 en 8 bits → 12₁₀ = 00001100₂ → con signo positivo: 00001100
Complemento a 2 — Rangos de enteros
Fórmula del rango: [−2^(n−1), 2^(n−1) − 1]
TipoBitsRango
short (entero corto)16[−2¹⁵, 2¹⁵ − 1]
int (entero)32[−2³¹, 2³¹ − 1]
long (entero largo)64[−2⁶³, 2⁶³ − 1]
Punto flotante — IEEE 754
Estructura general
Un número en punto flotante se descompone en tres partes:
Signo (S) — indica positivo (0) o negativo (1)
Exponente (E) — determina el orden de magnitud; permite que el punto "flote"
Mantisa / Fracción (M) — contiene los dígitos significativos, representa la precisión
PrecisiónTotal bitsSignoExponenteMantisa
Simple (float) 32 bits 1 bit 8 bits 23 bits
Doble (double) 64 bits 1 bit 11 bits 52 bits
OVERFLOW
Desbordamiento
El exponente es demasiado grande para ser representado en su campo. El número supera el máximo representable.
UNDERFLOW
Subdesbordamiento
El exponente negativo es menor al mínimo representable (más negativo). El número es demasiado pequeño en magnitud.
Errores en métodos numéricos
Tipos de error
Error de truncamiento: reemplazar un proceso infinito por uno finito (cortar una serie).
Error de redondeo: limitar cifras decimales (usar 3.1416 en lugar de π).
Error acumulado: se suma en operaciones sucesivas.

Precisión: qué tan cerca están múltiples resultados entre sí.
Exactitud: qué tan cerca está el resultado del valor verdadero.

Procedimientos y ejemplos de conversión entre sistemas

Decimal → Binario (entero)
Método: divisiones sucesivas entre 2
1. Divide N entre 2, anota cociente y residuo.
2. Repite con el cociente hasta que sea 0.
3. Lee los residuos de abajo hacia arriba.

25₁₀ → binario

  • 25 ÷ 2 = 12r=1
  • 12 ÷ 2 = 6r=0
  • 6 ÷ 2 = 3r=0
  • 3 ÷ 2 = 1r=1
  • 1 ÷ 2 = 0r=1
25₁₀ = 11001₂

44₁₀ → binario

  • 44 ÷ 2 = 22r=0
  • 22 ÷ 2 = 11r=0
  • 11 ÷ 2 = 5r=1
  • 5 ÷ 2 = 2r=1
  • 2 ÷ 2 = 1r=0
  • 1 ÷ 2 = 0r=1
44₁₀ = 101100₂
Binario → Decimal (entero)
Fórmula: suma de potencias de 2
N = an×2n + an-1×2n-1 + … + a1×21 + a0×20
Numerar los bits de derecha a izquierda desde 0.

111011₂ → decimal

  • 1×2⁵ = 32
  • 1×2⁴ = 16
  • 1×2³ = 8
  • 0×2² = 0
  • 1×2¹ = 2
  • 1×2⁰ = 1
111011₂ = 59₁₀

111100₂ → decimal

  • 1×2⁵ = 32
  • 1×2⁴ = 16
  • 1×2³ = 8
  • 1×2² = 4
  • 0×2¹ = 0
  • 0×2⁰ = 0
111100₂ = 60₁₀
Decimal → Binario (fracción)
Método: multiplicaciones sucesivas por 2
1. Multiplica la parte fraccionaria por 2.
2. La parte entera del resultado (0 o 1) es el siguiente bit.
3. Repite con la parte fraccionaria restante.
4. Lee los bits de arriba hacia abajo.

0.6875₁₀ → binario

  • 0.6875×2 = 1.375→1
  • 0.375×2 = 0.75→0
  • 0.75×2 = 1.50→1
  • 0.50×2 = 1.00→1
0.6875₁₀ = 0.1011₂

0.4375₁₀ → binario

  • 0.4375×2 = 0.875→0
  • 0.875×2 = 1.75→1
  • 0.75×2 = 1.50→1
  • 0.50×2 = 1.00→1
0.4375₁₀ = 0.0111₂
Representación infinita
Algunos decimales no tienen representación binaria exacta. Ejemplo: 0.65₁₀ = (0.1010011…)₂ — secuencia infinita periódica. La computadora almacena una aproximación.
Binario → Decimal (fracción)
Fórmula: suma de potencias negativas de 2
F = b₁×2⁻¹ + b₂×2⁻² + b₃×2⁻³ + …
Donde b₁ es el primer bit después del punto, b₂ el segundo, etc.

0.1111₂ → decimal

  • 1×2⁻¹ = 0.5
  • 1×2⁻² = 0.25
  • 1×2⁻³ = 0.125
  • 1×2⁻⁴ = 0.0625
0.1111₂ = 0.9375₁₀

0.1101₂ → decimal

  • 1×2⁻¹ = 0.5
  • 1×2⁻² = 0.25
  • 0×2⁻³ = 0
  • 1×2⁻⁴ = 0.0625
0.1101₂ = 0.8125₁₀
